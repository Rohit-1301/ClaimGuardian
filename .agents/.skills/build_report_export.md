# Skill: Build ClaimGuardian Report Export System

ROLE:
You are a backend product engineer.

MISSION:
Create downloadable professional reports for claim analysis results.

Purpose:
Allow users to save/share pre-submission validation outcomes.

---

# REPORT TYPES

1. PDF (mandatory)
2. JSON export (recommended)
3. CSV summary (optional)

---

# PDF CONTENT REQUIREMENTS

Header:
- ClaimGuardian logo/title
- report generated timestamp
- report ID

User Section:
- claimant name (masked if needed)
- claim reference ID

Documents Uploaded:
- policy
- bill
- discharge summary
- others

Analysis Results:
- risk score
- numeric score
- issues list
- severity labels
- suggestions

Extracted Key Fields:
- hospital
- treatment dates
- amount

Disclaimer:
This report is guidance only and not insurer approval.

Footer:
- generated by ClaimGuardian

---

# STYLING

Use professional clean layout:
- spacing
- headings
- tables
- readable fonts

---

# BACKEND FILES

backend/services/report_service.py

---

# API ROUTES

GET /api/reports/{claim_id}/pdf  
GET /api/reports/{claim_id}/json

---

# STORAGE OPTIONS

1. Generate on demand
2. Save metadata in DB
3. Optional file storage in Supabase Storage

---

# QUALITY RULES

1. Handle missing values gracefully
2. Never expose secrets
3. Mask PHI where required
4. Filename should be safe

Example:

ClaimGuardian_Report_1234.pdf
